# NexusRecon Report File Index

Every campaign writes its output to:

```
campaigns/<client>/<engagement_id>/<campaign_id>/reports/
```

---

## Report Files

| File | Format | Generated by | Contents |
|------|--------|--------------|----------|
| `master_report.md` | Markdown | `_master_report()` | **The cohesive narrative deliverable.** Single document combining cover, executive brief, top threads, conditional attack-surface sections, evidence & provenance, recommendations, and a dynamic appendix linking the other reports. The file an operator hands a client when they want one. |
| `executive_summary.md` | Markdown | `_executive_summary()` | 1-page summary: scope, key findings, top 3 actions |
| `full_report.md` | Markdown | `_full_report()` | Complete findings with methodology, tool output, MITRE mapping |
| `top_threads.md` | Markdown | `_top_threads()` | Top 10 ranked attack threads (score + category + rationale) |
| `asset_inventory.md` | Markdown | `_asset_inventory()` | All discovered subdomains, emails, IPs, cloud assets |
| `asset_inventory.json` | JSON | `_asset_inventory()` | Machine-readable asset list |
| `asset_inventory.csv` | CSV | `_asset_inventory()` | Spreadsheet-compatible asset list |
| `phishing_package.md` | Markdown | `_phishing_package()` | Target emails, pretext hooks, DMARC gaps, per-employee bundles |
| `cloud_posture.md` | Markdown | `_cloud_posture()` | M365 federation, public S3/GCS buckets, cloud exposure |
| `attack_surface.md` | Markdown | `_attack_surface()` | Likelihood × impact matrix, PRE-ATT&CK mapped |
| `vuln_correlation.md` | Markdown | `_vuln_correlation()` | CVE/KEV findings matched to fingerprinted technologies |
| `people_map.md` | Markdown | `_people_map()` | Org chart inference, executive targets, HUMINT leads |

## Key Files Explained

### `master_report.md`

The single cohesive narrative report — one document an operator can hand a
client that explains everything found and what it means. Sits between the
one-page executive summary and the file sprawl of individual reports.

Structure (sections only appear when they have content):

1. **Snapshot** — campaign id, scope hash, findings totals, costs (always)
2. **Executive Brief** — 200-400 words LLM-synthesized prose (always)
3. **Top Threads to Pull** — full content of `top_threads.md` embedded inline
4. **Attack Surface at a Glance** — Identity, Cloud, Code & Secrets, Network
   (subsections appear only when their underlying state is populated; the
   parent heading appears only if at least one subsection has content)
5. **Identified Personas** — only if any emails harvested
6. **Vulnerability Correlation** — only if enriched CVEs or nuclei findings
7. **Harvested Credentials** — only if `--validate-creds` produced records
8. **Pretext & HUMINT** — only if pretext intel collected
9. **Evidence & Provenance** — scope hash, audit chain, cost ledger (always)
10. **Recommendations** — 5-10 prioritized actions, LLM-synthesized (always)
11. **Appendix: Deeper Reading** — dynamic links to the other reports based
    on which files actually exist (always)

Generated by the `master_reporter` agent (one LLM call per campaign, typical
cost < $1.00). The agent is in `_SKIP_FINDINGS_JSON_AGENTS` so its output is
pure markdown, not a FINDINGS_JSON block.

### `executive_summary.md`

Written for the client's CISO or executive sponsor. Includes:
- Engagement scope and methodology
- Critical and high findings (first 5)
- Three immediate recommended actions

### `top_threads.md`

The attack-planning centrepiece. Each thread has:
- `score` (0–1): `(cvss/10) × max(epss, 0.05) × kev_boost × msf_boost`
- `category`: `cve`, `credential`, `breach`, `cloud`, `infra`, `social`
- `title`, `description`, `affected_assets`, `suggested_next_steps`

Header: **`# Top 10 Threads to Pull`**

### `harvested_credentials.md`

**Sensitive — treat as Secret.** Contains credentials extracted from exposed config
files, `.env` files, and public code repositories. Values are masked
(e.g., `AKIA***KE`) and hashed (SHA-256) — the raw credential is never written to
disk. The file opens with:

> ⚠ This file contains real credentials. Treat as Secret. Rotate before sharing.

### `findings.json`

Machine-readable array of `Finding` objects:

```json
[
  {
    "id": "nx-...",
    "title": "Exposed AWS credentials in .env file",
    "severity": "critical",
    "confidence": 0.95,
    "category": "credential",
    "description": "...",
    "affected_assets": ["example.com"],
    "evidence": ["https://example.com/.env"],
    "mitre_techniques": ["T1552.001"],
    "citations": ["crtsh", "infra_probe"]
  }
]
```

### `maltego_export.csv`

Import via Maltego → Import Entities from CSV. Columns:
`entity_type`, `value`, `source`, `confidence`, `notes`

## Conditional Files

Some files are only generated when there is data to populate them:

| File | Condition |
|------|-----------|
| `harvested_credentials.md` | Only if `harvested_credentials` list is non-empty |
| `report.pdf` | Only if `weasyprint` is installed |
| `phishing_package.md` | Always generated; richer when emails + pretext data are present |
| `vuln_correlation.md` | Always generated; richer when `vuln_intel.enriched_cves` is non-empty |

---

## Return Value of `generate_all()`

`ReportEngine.generate_all(state)` returns a `Dict[str, str]` mapping report name to
absolute file path. If a report was not generated (e.g., `report.pdf` without
`weasyprint`), the value is `None`.

```python
paths = engine.generate_all(state)
# {
#   "executive_summary": "/path/to/reports/executive_summary.md",
#   "harvested_credentials": "/path/to/reports/harvested_credentials.md",
#   "report_pdf": None,   # weasyprint not installed
#   ...
# }
```

## Customising Reports

Reports are generated by pure Python string templating in `nexusrecon/reports/engine.py`.
To modify a report's format:

1. Find the private method (e.g., `_top_threads()`)
2. Edit the `lines` list — it is plain Python, no template language needed
3. Run `nexusrecon smoke` to verify the output format is still valid
